# Требования нормоконтроля документов — SmartDoc & Code Review

> **Область применения.** Данный документ описывает требования к оформлению, реализованные в модуле нормоконтроля `.docx` проекта SmartDoc & Code Review. На текущем этапе модуль поддерживает **только** требования **АГУ им. В.Н. Татищева**, факультет цифровых технологий и кибербезопасности (ФЦТиК), кафедра информационных технологий и кибербезопасности. Требования других факультетов и вузов не реализованы — для их добавления предусмотрен механизм пользовательских правил.

---

## Обоснование выбора стандарта

Внутри одного университета (АГУ) существуют принципиально разные подходы к оформлению: инженерно-технический (ФЦТиК — ЕСКД, рамки, штампы, нормоконтроль) и академический (например, химический факультет — ГОСТ 7.32, без рамок, акцент на библиографическую культуру). Шаблон одного факультета нельзя использовать для другого — работа будет отклонена по формальным признакам.

Проект SmartDoc ориентирован на направление 09.03.02 «Информационные системы и технологии» (ФЦТиК), поэтому за основу взят инженерно-технический подход с ориентацией на ЕСКД.

## 1. Архитектурные отличия

### Рамки и штампы

Самое крупное расхождение между двумя подходами.

**ФЦТиК [2]** использует рамки и штампы по ЕСКД (ГОСТ 2.104-2006): каждый лист содержит рамку с полями «Изм.», «Лист», «№ документа», «Подпись», «Дата» и основную надпись с указанием ролей (разработчик, проверяющий, нормоконтролёр). Это стандарт конструкторской документации, характерный для инженерных и IT-направлений.

**Химфак [1]** использует стандартное академическое оформление без рамок — чистый лист А4 с полями, нумерация страниц вверху справа, формат научной работы по ГОСТ 7.32-2001.

### Базовые стандарты

| Подход | Базовый стандарт |
|---|---|
| ФЦТиК | ЕСКД (ГОСТ 2.104-2006) |
| Химфак | ГОСТ 7.32-2001 «Отчёт о НИР», ГОСТ 7.80-2000 |

## 2. Параметры форматирования

Ниже перечислены параметры, которые отличаются между подходами. Столбец «SmartDoc» показывает, что именно проверяет текущая версия модуля нормоконтроля.

| Параметр | Химфак [1] | ФЦТиК [2] | SmartDoc (текущая реализация) |
|---|---|---|---|
| Формат бумаги | А4 | А4 (с рамкой ЕСКД) | А4 |
| Кегль основного текста | 14 | 14 (предположительно) | **14** — проверяется |
| Шрифт | Times New Roman | Times New Roman | **Times New Roman** — проверяется |
| Межстрочный интервал | 1,5 | Не указан явно | **1,5** — проверяется |
| Выравнивание | По ширине | По ширине | **По ширине** — проверяется |
| Поля (лево / право / верх / низ) | 30 / 15 / 20 / 20 мм | Определяются рамкой ЕСКД | **30 / 15 / 20 / 20 мм** — проверяется |
| Абзацный отступ | 1,5 см | Не указан явно | **1,25 см** — проверяется |
| Нумерация страниц | Арабские, вверху справа, сквозная | В штампе ЕСКД | Наличие нумерации — проверяется |

## Перспективы расширения

Архитектура SmartDoc предусматривает механизм пользовательских правил, что позволяет в будущем добавить профили для других факультетов АГУ и других вузов без переработки ядра системы. Для расширения необходимы методические указания соответствующих кафедр.
